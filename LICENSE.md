## 1. 定義

**1.1** 「專案」指由 o.O Labs 開發並根據本許可證分發的 Unreal Engine 專案,包括所有源代碼、資產、文檔及相關材料。

**1.2** 「o.O Labs」指專案的版權所有人及開發方,為依香港特別行政區法律註冊或運營的法律實體。

**1.3** 「衍生作品」指基於專案創建的任何修改版本、改編版本或其他作品,其客戶端具有與原始專案不同的行為特徵,包括但不限於非官方構建、自定義客戶端或經過修改的實現。

**1.4** 「您」或「被許可方」指根據本許可證使用、修改或分發專案或衍生作品的任何個人或法律實體。

**1.5** 「代碼資產」指需要編譯、解釋或執行的所有源代碼文件、腳本、著色器及其他可執行組件,包括但不限於 C++、Blueprint、Python 腳本及格式化配置文件(如 JSON、XML)。

**1.6** 「非代碼資產」指無需編譯或解釋即可使用的所有資產,包括但不限於 3D 模型、紋理、音頻文件、圖像、動畫及其他媒體內容。

**1.7** 「商業軟件」指任何受專有許可證約束的第三方軟件、插件、庫或組件。

**5.1 創建與分發**

5.1.1 您可以創建、修改和分發專案的衍生作品,前提是遵守本許可證的所有條款以及 Unreal EULA。

5.1.3 衍生作品的創建者對該衍生作品承擔全部法律責任。

**5.2 合理使用與知識產權**

5.2.1 衍生作品可以以絕對合理使用的方式使用 o.O Labs 的知識產權,包括但不限於商標、標誌和品牌元素,前提是此類使用:
- 純粹是描述性的和事實性的
- 不暗示 o.O Labs 的贊助、背書或附屬關係
- 不會在消費者中造成混淆

5.2.2 衍生作品必須清楚地將自己標識為非官方作品,與 o.O Labs 沒有附屬關係。

**5.3 禁止背書**

5.3.1 衍生作品不得以任何方式明示或暗示 o.O Labs 對該作品背書、贊助或批准。

5.3.2 衍生作品必須在其分發、營銷材料和用戶界面中包含顯著的免責聲明,說明:
- 該作品是非官方的衍生作品
- o.O Labs 不為該作品背書或承擔責任
- 該作品由獨立的第三方維護

**5.4 商業組件集成**

5.4.1 專案本身不包含或主動支持任何商業軟件的集成。

5.4.3 衍生作品創建者對確保遵守所有集成的商業軟件的許可證條款承擔全部責任。

5.4.4 o.O Labs 對衍生作品中使用的任何商業軟件不提供支持、保證或責任。

**5.5 透明度要求(建議)**

5.5.1 強烈建議衍生作品創建者在其文檔中列出:
- 所有對原始專案進行的自定義修改
- 所有集成的商業組件及其相應的許可證
- 任何可能影響用戶體驗或法律義務的重大變更

5.5.2 此建議主要是為了用戶的便利和透明度,但不構成強制性要求。

**5.6 歸屬聲明(強制要求)**

5.6.1 所有衍生作品必須在以下位置包含清晰可見的歸屬聲明:
- 主要文檔或 README 文件
- 應用程序的「關於」或「致謝」界面(如適用)
- 任何分發包或安裝程序的顯著位置

5.6.2 歸屬聲明必須採用以下格式:

「本項目基於 o.O Labs Unreal Project 商業許可證(日期 [YYYY-MM-DD] 版本 [X.X])所賦予的權利進行 [開發/修改/分發]。」

其中:
- [YYYY-MM-DD] 應替換為您獲得授權時本許可證的生效日期
- [X.X] 應替換為本許可證的版本號
- [開發/修改/分發] 應根據實際情況選擇適當的動詞

5.6.3 歸屬聲明必須使用與周圍文本相同或更大的字體大小,並且不得以任何方式被隱藏、模糊或難以發現。
